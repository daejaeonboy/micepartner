# Admin Content Management

현재 관리자 페이지에서는 아래 작업을 처리할 수 있습니다.

- 관리자 회원가입 / 이메일 로그인 / Google 로그인
- 홈페이지 핵심 텍스트 수정
- 페이지별 구조 데이터(JSON) 수정
- 문의 목록 조회 및 상태 변경

## 1. 관리자 인증

관리자 인증은 SQLite 기반 관리자 계정과 세션으로 처리합니다.

Google 로그인을 사용하려면 아래 환경변수를 설정합니다.

- `GOOGLE_CLIENT_ID`
- `VITE_GOOGLE_CLIENT_ID`

설정 파일 예시는 아래를 참고하면 됩니다.

- `.env.example`

## 2. 관리자 로그인 위치

- 관리자 페이지 경로: `/admin`

로그인 후 가능한 작업:

- 관리자 회원가입 / 이메일 로그인 / Google 로그인
- 메인 페이지 문구 수정
- 서비스 페이지 문구 수정
- 포트폴리오 페이지 문구 수정
- 자료실 페이지 문구 수정
- 회사소개 페이지 문구 수정
- 문의 페이지 문구 수정
- 푸터 문구 수정
- 페이지별 JSON 구조 데이터 수정
- 문의 목록 상태/메모 관리

## 3. 문구 저장 방식

- 사이트 문구와 구조 데이터는 SQLite DB에 저장됩니다.
- 문의 데이터, 관리자 계정, 관리자 세션과 같은 DB 파일을 사용합니다.

관련 파일:

- `server/index.js`
- `server/db.js`
- `server/defaultSiteCopy.js`
- `src/context/SiteContentContext.tsx`
- `src/pages/AdminPage.tsx`

## 4. 현재 관리자에서 수정되는 범위

현재는 페이지의 핵심 텍스트와 구조 데이터를 함께 관리자에서 수정할 수 있습니다.

- 상단 소개 제목/설명
- 섹션 제목/설명
- CTA 문구
- 푸터 설명
- 카드 목록
- 포트폴리오 상세 본문 / 이미지 / 갤러리
- 공지 상세 본문 / 이미지 / 첨부 링크
- 자료 상세 본문 / 다운로드 링크 / 버전 정보

아직 코드에서 직접 관리하는 항목:

- 아이콘 매핑 문자열 목록
- 실제 이미지 업로드 스토리지

이 영역은 다음 단계에서 스토리지 연동과 폼 세분화로 확장할 수 있습니다.
