# Gemini Design Brief

작성일: 2026-03-10

아래 내용은 Gemini CLI를 비대화식 모드로 호출해 받은 회사 홈페이지 디자인 브리프 초안이다.

실행 명령 형태:

```powershell
gemini -p "<디자인 브리프 요청>" 
```

## 요약

### 1. 브랜드 인상

- 전문성과 실행력을 강조한다.
- 공공기관과 기업 고객이 모두 신뢰할 수 있는 안정감을 준다.
- 복잡한 장식보다 명확한 정보 구조와 성과 중심 표현을 우선한다.

### 2. 컬러 방향

- Primary: Deep Navy `#1A2B4B`
- Secondary: Slate Gray `#4A5568`
- Accent: Royal Blue `#2563EB`
- Base: White / Light Gray `#F8FAFC`

### 3. 타이포 방향

- `Pretendard` 또는 `Noto Sans KR`
- 제목은 Bold, 본문은 충분한 행간 확보
- 기본 좌측 정렬, 메인 섹션만 중앙 정렬 혼용 가능

### 4. 섹션 레이아웃 원칙

- 여백을 충분히 둬서 읽히는 콘텐츠 구조를 만든다.
- 반복 정보는 카드형 UI로 통일한다.
- 섹션 간에는 배경 명도 차이로 구분감을 준다.

### 5. 포트폴리오 페이지 원칙

- 상단에 카테고리 필터를 둔다.
- 프로젝트명, 기간, 핵심 성과를 먼저 보여준다.
- 가능하면 수치나 키워드로 결과를 시각화한다.

### 6. 모바일 원칙

- 주요 액션은 손이 닿기 쉬운 위치에 둔다.
- 본문 폰트는 16px 미만으로 낮추지 않는다.
- 이미지는 1단 배열과 지연 로딩 중심으로 구성한다.

## 비고

- 현재 사용자 요청에 따라 디자인 자체는 직접 적용하지 않고 구조 작업만 진행한다.
- 이후 실제 리디자인 단계에서 이 문서를 기준 참고안으로 사용할 수 있다.
